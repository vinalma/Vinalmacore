# Clientes

Cada cliente fica isolado em sua pasta.

## Estrutura padrão

```
clients/
  <nome-cliente>/
    briefing.md       ← dados do cliente
    01-strategy/      ← funil, posicionamento
    02-traffic/       ← campanhas ativas
    03-creative/      ← briefs e criativos
    04-results/       ← relatórios
```

## Regras

- Clientes não compartilham contexto entre si
- Crie a pasta apenas quando o cliente for ativo
- Mova para `archive/clients/` ao encerrar
